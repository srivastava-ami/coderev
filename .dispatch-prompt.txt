## Task: v110_rule_yaml

## What to build

Add the first real coderev-owned semgrep YAML rule: security.secret_fallback_literal.
This covers the destructuring default form and multi-line assignments that the Go
line scanner cannot detect. Delete the placeholder.yaml added by Task B.

## Why

The Go treesitter scanner covers dot-notation and bracket-notation forms.
Semgrep's AST-aware matching covers:
  const { SIGNING_KEY = 'dev-key' } = process.env
  const { JWT_SECRET = 'x', PORT = '3000' } = process.env
These require understanding destructuring assignment structure, not line patterns.

## Exact implementation spec — follow this precisely

The spec below contains the EXACT code to write. Copy it verbatim.
Do not invent alternative implementations.

### security_yaml

Write this exact content to internal/adapters/semgrep/rules/security.yaml.
Note: $VAL uses '.+' which requires a non-empty match — empty fallback does not fire.

  rules:
    - id: security.secret_fallback_literal
      languages: [typescript, javascript]
      severity: ERROR
      message: >
        Env var '$KEY' has a hardcoded literal fallback. If the variable is unset
        in a deployed environment this uses a known, source-visible value.
        Fail closed (throw) instead of defaulting.
      metadata:
        cwe: "CWE-798"
        owasp: "A07:2021"
        source: coderev
      pattern-either:
        - pattern: process.env.$KEY ?? "$VAL"
        - pattern: process.env.$KEY || "$VAL"
        - pattern: process.env[$KEY] ?? "$VAL"
        - pattern: process.env[$KEY] || "$VAL"
        - pattern: |
            const {..., $KEY = "$VAL", ...} = process.env
      metavariable-regex:
        $KEY: '(?i)(secret|password|passwd|token|api[_-]?key|private[_-]?key|client[_-]?secret|credential|jwt|signing[_-]?key|encryption[_-]?key)'
        $VAL: '.+'
      paths:
        exclude:
          - "**/*.test.ts"
          - "**/*.spec.ts"
          - "**/*.test.tsx"
          - "**/*.spec.tsx"
          - "**/*.test.js"
          - "**/*.spec.js"
          - "**/__tests__/**"
          - "**/e2e/**"
